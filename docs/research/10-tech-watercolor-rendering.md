# Technical Research: Watercolor / Storybook Rendering in Unity (Small Team, 2026)

Research date: July 2026. Scope: pipeline choice (URP vs Built-in, Unity 6 vs 2022.3), concrete watercolor NPR techniques with sources, vegetation and atmosphere, performance for dense small scenes, buy-vs-build asset list, and a recommended rendering stack for the Storybook Isometric Camera MVP scene.

## Key Takeaways

- **The pipeline question answered itself in June 2026:** Unity deprecated the Built-in Render Pipeline in Unity 6.5 and put HDRP in maintenance mode; URP is the sole recommended pipeline for new projects. Unity 2022.3 LTS is already end-of-life (updates now Enterprise/Industry-only since mid-2025). Staying on 2022.3 + Built-in is building on two dead platforms. Move to **Unity 6.3 LTS + URP now**, while the project has almost no code to migrate.
- **The single biggest lesson from shipped watercolor games (Dordogne): the look lives in hand-painted textures, not in a clever shader.** Dordogne's art director explicitly warns that mocked-up/animated watercolor shaders "usually look fake" — they shipped real scanned watercolors on simple shaders with white rim light.
- **Eastshade's painterly trick is nearly free:** a black-and-white paint-splat mask baked into texture mipmaps so distant surfaces dissolve into paint strokes — cheap, works in any pipeline, zero runtime cost.
- **A real-time watercolor look decomposes into ~6 named effects** (edge darkening, pigment turbulence, granulation, paper distortion, bleeding, gaps & overlaps) with published real-time implementations — you cherry-pick 2-3 of them, subtly, rather than implementing all.
- **Kuwahara/anisotropic filtering is a trap as a core look** for a detail-pillar game: it destroys the small handcrafted detail you're selling. Use it (if at all) as a distant-background-only or photo-mode filter.
- **Shader Graph in URP is fully viable for custom toon/ramp lighting** via Custom Function nodes; Cyanilux maintains a free, Unity-6-ready custom lighting sub-graph library, so you don't need to hand-write HLSL from scratch.
- **Vegetation: buy, don't build.** Staggart's Stylized Grass Shader (~$23-35, Unity 6 supported) gives wind, player-bend interaction and terrain color blending out of the box; Unity 6's GPU Resident Drawer (Forward+ only) collapses tens of thousands of foliage instances into ~hundreds of batches.
- **Time-of-day and seasons are a color-grading problem, not a lighting problem:** blend 3D LUTs per time-of-day via URP volumes (a 32³ LUT is standard; sunset needs its own LUT, not a day↔night lerp) plus swapped color/ramp palettes per season.
- **Total buy-list is under ~$250** for grass, water, fog, and optionally outlines/toon base — trivially worth it for a 1-3 person team.

## 1. Pipeline: Unity 6 LTS + URP (the 2026 reality)

**Unity 2022.3 is done.** 2022.3 LTS support ended around May 2025, and continued updates were then restricted to Enterprise/Industry licenses ([Unity discussions: "2022LTS is EOL"](https://discussions.unity.com/t/2022lts-is-eol/1644976), [endoflife.date/unity](https://endoflife.date/unity)). The project's current 2022.3.50f1 install receives no fixes.

**Built-in RP is deprecated.** Unity's official "Render Pipelines strategy for 2026" (announced with Unity 6.5, released June 15, 2026) deprecates BIRP; it remains available through Unity 6.7 LTS with support guaranteed through 2028 (2029 for Enterprise/Industry), but Unity explicitly tells new projects to use URP ([Unity strategy page](https://unity.com/topics/render-pipelines-strategy-for-2026), [80.lv summary](https://80.lv/articles/unity-unveils-2026-render-pipelines-strategy), [MaxKill Studios migration analysis](https://www.maxkillstudios.com/learn/birp-deprecated-migrate-urp-or-wait)). **HDRP is in maintenance mode** — no new features, only stability fixes ([GameFromScratch: "The Unity HDRP is Dead"](https://gamefromscratch.com/the-unity-hdrp-is-dead/)). URP is receiving the investment: real-time GI, physical sky, screen-space reflections, faster build times ([80.lv](https://80.lv/articles/unity-unveils-2026-render-pipelines-strategy)).

**Unity 6.3 LTS** shipped December 2025 ([CG Channel](https://www.cgchannel.com/2025/12/unity-6-3-lts-is-out-see-5-key-features-for-cg-artists/)); URP and HDRP now share a compiler/API there ([Unity blog](https://unity.com/blog/unity-6-3-lts-is-now-available)). **Render Graph** is the default rendering API for custom URP passes in Unity 6; a compatibility mode exists for old `ScriptableRenderPass` code but documentation for retrofitting upgraded projects is thin ([Unity discussions](https://discussions.unity.com/t/implementing-render-graph-into-an-upgraded-urp-project-to-unity-6/947237), [URP 17 upgrade guide](https://docs.unity3d.com/6000.3/Documentation/Manual/urp/upgrade-guide-unity-6.html)). Starting fresh on Unity 6 avoids that retrofit entirely — write all custom passes (edge darkening, paper overlay, roof-fade masks) against Render Graph from day one.

**Upgrade cost for this project: near zero.** Pre-production, no URP/Cinemachine installed, no shader inventory. This is the cheapest the migration will ever be. The costs that would exist in a mature project (rewriting Built-in surface shaders, custom passes, asset incompatibilities) don't apply yet — and Asset Store content is being pushed to be "URP-compatible by default" ([80.lv](https://80.lv/articles/unity-unveils-2026-render-pipelines-strategy)), so buying stylized assets increasingly assumes URP.

**Verdict: Unity 6.3 LTS, URP, Forward+ rendering path, Render Graph native (no compatibility mode).** Forward+ is required for GPU Resident Drawer (see §5) and removes the 8-lights-per-object limit for lantern-lit evening scenes.

## 2. What "watercolor" means, technically

Academic real-time watercolor (Montesdeoca et al., *Art-directed watercolor stylization of 3D animations in real-time*, 2017) decomposes the medium into named, separately-implementable effects ([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0097849317300316)); the commercial Flair/Artineering implementation uses the same catalogue ([artineering.io/styles/watercolor](https://artineering.io/styles/watercolor)):

| Effect | What it is | Real-time approach |
|---|---|---|
| **Edge darkening** | Pigment migrates to stroke edges as water dries, leaving dark rims | Difference-of-Gaussians or Sobel on color/depth; darken (not blacken) discontinuities |
| **Pigment turbulence** | Blotchy uneven density inside washes | Low-frequency noise modulating albedo/saturation |
| **Granulation** | Pigment settling into paper tooth | Paper texture multiplied in, strongest in dark/shadow areas |
| **Paper distortion** | Wobbly contours from wet paper | Small UV displacement of the scene or of edges by paper-noise |
| **Color bleeding** | Wet-in-wet colors flowing into neighbors | Hybrid object/image-space blur toward neighbors — the most expensive, least necessary effect |
| **Gaps & overlaps** | Paint not quite reaching, or crossing, boundaries | Offset/eroded masks at silhouettes |

Two implementation notes recur across sources: watercolor edge darkening is the single highest-impact effect (the classic "coffee-stain" ring — darker rims, lighter centers, sharp falloff: [AleksLitynski/WatercolorShader](https://github.com/AleksLitynski/WatercolorShader)); and screen-space paper texture creates the **"shower door" artifact** (the world appears to slide behind a fixed glass texture) — mitigated by anchoring texture to world/object space (subtract object origin from the sample position) or by Flair-style "canvas advection" that drags the paper with object motion ([artineering](https://artineering.io/styles/watercolor), [Patch Quest paper shading breakdown](https://lycheelabs.net/hood-paper-shading/)). With a mostly-static isometric camera and gentle dollies, this artifact is far less visible than in a first-person game — a genuine advantage of the Storybook Camera.

## 3. Case studies (what shipped games actually did)

**Dordogne (Unity, 2023) — the real-watercolor ceiling.** Art director Cédric Babouche painted scenes in real ink + watercolor on paper, scanned them, and camera-mapped them onto rough 3D in Blender — 2-3 camera projections for simple scenes, 5-7 plus textured props for complex ones; lighting and shadows were painted into the illustrations; the shader itself was "very simple" — watercolor textures plus a white rim light. Key quotes: animated watercolor shaders "usually look fake," and "paper and traditional techniques are time-saving" vs. faking watercolor in Photoshop ([Game Developer Q&A](https://www.gamedeveloper.com/design/q-a-hand-painting-the-watercolor-world-of-i-dordogne-i-)). The cost: severe movement restrictions and edge-blending hacks — acceptable for a narrative side-scroller, **not** for a free-roaming isometric game. Lesson to steal: paint the texture, keep the shader humble. Lesson to avoid: full camera-mapping.

**Eastshade (Unity, Built-in, 2019) — painterly on a budget.** Danny Weinbaum's signature trick: a black-and-white mask of paint splatters baked into the **mipmap chain** — the mask acts as alpha in low mips so distant surfaces resolve as loose paint blobs, with detail "painting itself in" as you approach; described as "cheap and easy to pull off inside of Unity" ([Game Developer](https://www.gamedeveloper.com/design/turning-painting-into-a-game-mechanic-in-the-gorgeous-i-eastshade-i-)). His foliage/perf writeup is equally valuable: draw calls (not polycount) are the budget; one 2048×2048 atlas for all foliage; trees authored so LOD1 is a whole-canopy simplification; a "hex grid" system statically combines only LOD1 meshes; Light Probe Proxy Volumes give canopies a lighting gradient ([eastshade.com/foliage-optimization-in-unity](https://eastshade.com/foliage-optimization-in-unity/)).

**Ni no Kuni (Level-5) — the toon-ramp storybook.** Visible shading bands around model contours to mimic animation cels, outline rendering including interior lines (cheekbones, cloth folds) computed at render time, and — critically — **unified deep background colors** so environments don't read as "a mass of polygons," plus color-tone matching between real-time scenes and hand-drawn cutscenes ([Wikipedia](https://en.wikipedia.org/wiki/Ni_no_Kuni:_Wrath_of_the_White_Witch), [Cook and Becker on NnK2](https://www.cookandbecker.com/en/article/142/the-enchanting-art-style-of-ni-no-kuni-ii.html)). The relevant takeaway is palette discipline, not the hard cel bands — Brambly Hedge wants softer ramps.

**Snufkin: Melody of Moominvalley (Unity, 2024)** — your named camera reference — is Unity with hand-drawn, illustration-derived art ("a storybook come to life"); it proves the storybook look in Unity is an *art direction* achievement layered on ordinary rendering ([PCGamingWiki](https://www.pcgamingwiki.com/wiki/Snufkin:_Melody_of_Moominvalley), [Steam](https://store.steampowered.com/app/1808680/Snufkin_Melody_of_Moominvalley/)).

**Tiny Glade (custom Rust/Vulkan, 2024)** — your other reference — gets its softness from *lighting*, not textures: software ray-traced diffuse GI at 1 ray per 4×4 pixels against proxy geometry, encoded into spherical harmonics and denoised ([80.lv interview](https://80.lv/articles/exclusive-tiny-glade-developers-discuss-bevy-proceduralism-publishers-cozy-games), [dev's GPC talk teaser](https://x.com/h3r2tic/status/1813197502411092186)). You can't replicate that stack in Unity on a small team, but you can approximate the *result* in a handcrafted world: baked GI / Adaptive Probe Volumes, generous bounced ambient, nothing fully black. Softness is a lighting decision.

## 4. The shader toolbox (Shader Graph, URP)

**Ramp/gradient lighting — the foundation.** Shader Graph has no lit-toon master node, but Custom Function nodes calling `GetMainLight()` etc. are the standard pattern: [NedMakesGames' custom lighting tutorial](https://nedmakesgames.medium.com/creating-custom-lighting-in-unitys-shader-graph-with-universal-render-pipeline-5ad442c27276), [Daniel Ilett's Shader Graph custom lighting part](https://danielilett.com/2024-05-07-tut7-11-intro-to-shader-graph-part-7/), and — most useful — [Cyanilux's URP_ShaderGraphCustomLighting](https://github.com/Cyanilux/URP_ShaderGraphCustomLighting) sub-graphs with a maintained Unity 6 (URP 17.1+) branch, including toon ramp-texture main light and banded additional lights. Gotcha: sample ramp textures with **Sample Texture 2D LOD** (fixed mip), or distant objects shade differently as the ramp gets mipped ([Cyanilux](https://github.com/Cyanilux/URP_ShaderGraphCustomLighting)). For Brambly Hedge: a 2-3 stop ramp with wide soft transitions (10-25% smoothing, not hard cel steps), shadow color pushed toward desaturated cool violet/blue rather than black.

**Watercolor object shading — a proven Shader Graph recipe.** Cyanilux's watercolour experiments ([writeup](https://cyangamedev.wordpress.com/2020/10/06/watercolour-shader-experiments/)) document, node-by-node: triplanar-projected noise (avoids UV seams); shadow-edge darkening by sampling the shadow map with a noise-offset position and `step()`-ing attenuation so shadows get painterly dark rims and wobbly edges; soft diffuse via `saturate(dot(N,L) + 0.7)`; "patchy paint" via `abs(noise × strength − 0.3)`; subtle Fresnel edge darkening on curved surfaces; and a full-screen pass adding UV distortion (Simple Noise scale ~200, offset −0.5, × 0.01), Roberts-cross depth edge detection (×100 strength), and a **white vignette** — which instantly reads as "unpainted paper margin" and suits a storybook frame perfectly.

**Edge darkening / linework.** Depth+normals+color edge detection as a full-screen URP pass is thoroughly documented by [Alexander Ameye](https://ameye.dev/notes/edge-detection-outlines/) (Render Graph-native), with a commercial packaged version, Linework (€41.40, frequently ~€20; Unity 6 + 2022.3 URP, SRP-batcher/Render Graph aware) ([Asset Store](https://assetstore.unity.com/packages/vfx/shaders/linework-outlines-and-edge-detection-294140)). For watercolor, configure edges as *darkened warm pigment* a few shades deeper than the local color — not black ink — and displace edge UVs with paper noise for line wobble ([80.lv wobbly outline example](https://80.lv/articles/this-wobbly-outline-shader-will-make-your-3d-scenes-stylishly-hand-drawn)).

**Kuwahara / anisotropic filtering — use sparingly.** Reference implementations: [Garrett Gunnell's (Acerola) Unity post-processing repo](https://github.com/GarrettGunnell/Post-Processing) with an anisotropic Kuwahara shader, and [Dan Livings' URP oil-painting tutorial](https://www.danlivings.co.uk/blog/oil-painting-effect-shader-unity/) based on a 2019 multi-scale anisotropic Kuwahara paper. It's a strong oil-paint abstraction, but it eats exactly the miniature detail (stitching, jam jars, embroidery) that is a design pillar, and it shimmers under camera motion. Verdict: prototype it once for the record, expect to cut it; at most a weak, distance-masked pass on far background only.

**Paper texture overlay.** One multiply/soft-light layer of scanned cold-press paper over the final frame at low opacity (5-15%), strongest in midtones/shadows (granulation), anchored per §2 to avoid shower-dooring. Combine with Eastshade's mip-mask trick baked into environment albedo textures for a distance-based "dissolve into paint."

**Buyable watercolor shaders exist but are secondary:** [MK Watercolor Shader](https://assetstore.unity.com/packages/vfx/shaders/mk-watercolor-shader-305846) (per-object painted shading, from the MK Toon author), [Handpainted Shader and Watercolor Textures](https://assetstore.unity.com/packages/vfx/shaders/handpainted-shader-and-watercolor-textures-328667). Worth $20-40 to dissect for a reference vertical slice; the shipped game should own its hero shader.

## 5. Vegetation: oversized plants, grass, wind

Daniel Ilett's comparison of [six grass rendering techniques](https://danielilett.com/2022-12-05-tut6-2-six-grass-techniques/) is the best single overview: hand-placed meshes (max control, fine at small scale with batching), geometry/tessellation shaders (**avoid — unsupported/emulated on Metal, Quest, some consoles**), compute/procedural instancing (fastest, needs compute support), billboards/terrain details (distance filler), impostors (irrelevant at mouse scale). For a mouse-height camera inside grass, blades are *hero meshes*, which argues for mesh-based grass + instancing rather than billboard tricks.

- **Wind sway**: standard vertex-shader recipe — object-space position masked by height (roots pinned), offset by scrolling noise sampled with LOD; per-instance phase offset for desync; cross of wind direction with shape normal for natural bend ([NedMakesGames foliage shader](https://nedmakesgames.medium.com/creating-a-foliage-shader-in-unity-urp-shader-graph-5854bf8dc4c2), [Cyanilux GPU instanced grass breakdown](https://www.cyanilux.com/tutorials/gpu-instanced-grass-breakdown/)). Brambly-specific: at mouse scale wind should read as slow, heavy swaying of *individual* oversized stems (low frequency, higher amplitude, per-plant phase), not lawn-like shimmer.
- **[Stylized Grass Shader](https://assetstore.unity.com/packages/vfx/shaders/stylized-grass-shader-for-unity-2021-2023-143830) (Staggart, ~€23-35, URP, Unity 6 supported)**: wind with dynamic speed/direction, **bend/flatten interaction from trails, meshes and particles without a second camera**, hue variation, and a Color Map Renderer that blends grass color with the terrain beneath (docs: [terrain color blending](https://staggart.xyz/unity/stylized-grass-shader/sgs-docs/?section=blending-with-terrain-colors)). Mice pushing through parting grass is a core fantasy moment; this is the cheapest way to get it.
- **Scaling up**: Unity 6's **GPU Resident Drawer** (URP, **Forward+ only**, SRP Batcher on) auto-routes static meshes through BatchRendererGroup instancing — a 35,000-object foliage test collapsed to 128 batches ([Knights of U benchmark](https://theknightsofu.com/boost-performance-of-your-game-in-unity-6-with-gpu-resident-drawer/), [Unity manual](https://docs.unity3d.com/6000.0/Documentation/Manual/urp/gpu-resident-drawer.html)). Doesn't apply to skinned meshes/particles. [Nature Renderer](https://assetstore.unity.com/packages/tools/terrain/nature-renderer-2022-266664) ($99, per-Unity-version perpetual license) replaces the terrain detail system with cell-based instancing — only needed if you adopt Unity terrain at all; a handcrafted no-procgen hedge probably won't.

## 6. Lighting, time-of-day, seasons, atmosphere

- **Soft ambient base**: trilight ambient (sky/equator/ground gradient) + baked GI or Adaptive Probe Volumes; Light Probe Proxy Volumes for gradients through large canopies (Eastshade's trick, [source](https://eastshade.com/foliage-optimization-in-unity/)). Keep shadow floor lifted — nothing pure black (Ni no Kuni's "unified deep colors" discipline).
- **Time-of-day tinting via LUTs**: color grading bakes into a 3D LUT (32×32×32 typical — [Catlike Coding](https://catlikecoding.com/unity/tutorials/custom-srp/color-grading/)); URP Volumes blend LUT-carrying profiles by weight, with the caveat that all LUTs must share one size project-wide ([Unity post-processing docs](https://docs.unity3d.com/Packages/com.unity.postprocessing@3.0/manual/Color-Grading.html)). Because grading transforms are low-frequency, even 16-step LUTs suffice; and **sunset/dawn need their own LUTs — interpolating day↔night skips the golden hour** ([Shahriar Shahrabi's day-night LUT article](https://shahriyarshahrabi.medium.com/day-night-cycle-using-lut-in-fragment-shader-of-materials-80edaf26f655)). Plan: 4-5 keyed LUTs per season (dawn/day/golden/dusk/night), blended by game clock.
- **Seasonal palette swaps**: LUTs alone can't turn green leaves auburn without wrecking other hues. Combine per-season LUT sets with swapped albedo/color-map sets for foliage (the Stylized Grass colormap makes meadow-wide recoloring one texture swap) and a global "season" shader parameter lerping hue/saturation on vegetation materials.
- **Fog for diorama depth**: [Atmospheric Height Fog](https://assetstore.unity.com/packages/vfx/shaders/fullscreen-camera-effects/atmospheric-height-fog-optimized-fog-for-consoles-mobile-and-vr-143825) (BOXOPHOBIC, $20, frequently $10; Built-in/URP/HDRP) — height fog with directional light color and animated noise, volumetric-ish at trivial cost, explicitly not true volumetrics. For the storybook diorama: a low ground-mist layer at mouse height + gentle distance fog tinted to the sky LUT sells scale and separates foreground miniature detail from soft background. Add subtle depth-of-field on far background (Tiny Glade leans on DoF heavily) and the Cyanilux white vignette for the paper margin.
- **Dollhouse interiors / roof fade**: standard solutions are per-room roof renderer toggles or camera-facing dither/cutaway shaders (`Cull Off` cutaway examples: [Wikibooks Cg Cutaways](https://en.wikibooks.org/wiki/Cg_Programming/Unity/Cutaways)); in Unity 6 implement the fade as a Render Graph pass or a dithered crossfade material to keep SRP batching. Budget real time for this — it interacts with the outline/edge pass (edges must fade with the roof).

## 7. Performance for dense small scenes

Eastshade's ordering holds for hedgerow density: **draw calls first, fillrate second, polycount last** ([source](https://eastshade.com/foliage-optimization-in-unity/)). Practical stack for Unity 6 URP: SRP Batcher on (one hero uber-shader family keeps batching intact — a reason to build one Shader Graph, not twenty); GPU Resident Drawer for the thousands of static props/plants; texture atlasing for foliage (Eastshade: one 2048² atlas); overdraw discipline on grass cards (mouse-height camera looks *through* layers of alpha-tested grass — favor mesh blades over big transparent cards to dodge fillrate death); full-screen passes counted in a budget (edge pass + paper overlay + LUT grading + fog ≈ 4 cheap passes; Kuwahara would be the expensive fifth). Target: 60 fps at 1080p on a GTX 1060-class GPU and Steam Deck — the cozy audience skews toward modest hardware, and Tiny Glade set the expectation that gorgeous-soft runs on a 1060 ([HardForum thread](https://hardforum.com/threads/tiny-glade-beautiful-ray-traced-visuals-at-60fps-on-a-gtx-1060.2037236/)).

## Implications for Brambly Hedge: The Game

**1. Move to Unity 6.3 LTS + URP this month.** 2022.3 is EOL and Built-in is deprecated as of Unity 6.5 (June 2026). Every week on 2022.3 adds migration debt for zero benefit. Settings: URP, Forward+, Render Graph native, SRP Batcher on, GPU Resident Drawer on, linear color space. Skip HDRP entirely (maintenance mode). Take 6.3 LTS over 6.5 tech-stream for stability unless a 6.5 feature is load-bearing.

**2. Adopt the Dordogne doctrine: the watercolor lives in the textures; the shader whispers.** Pipeline: paint/scan real watercolor swatches and props (or paint with watercolor brush sets in Procreate/Krita for iteration speed), author albedo from them, keep lighting information out of albedo except soft AO. Jill Barklem's originals are dense cross-hatched watercolor — closer to Eastshade's "painted detail everywhere" than to loose washes, which is good news: it needs *painterly texture*, not simulated fluid.

**3. Build one hero uber-shader in Shader Graph** (vegetation/prop/character variants of the same graph to preserve SRP batching): soft 2-3 stop ramp via Cyanilux sub-graphs (transition smoothness ~0.15-0.25, shadows tinted cool violet-grey, never black), triplanar paper granulation multiplied into shadow regions, noise-offset shadow sampling + step for watercolor shadow rims, subtle Fresnel edge darkening, Eastshade mip-mask on environment textures, wind vertex sway on everything organic. Every effect gets a 0-1 art-direction slider; calibrate against a scanned Barklem plate side by side.

**4. Full-screen "storybook pass" (Render Graph):** Roberts-cross depth+normal edge *darkening* (warm dark pigment, noise-wobbled UVs, never black ink lines), low-opacity world-anchored paper overlay (5-15%, watch for shower-dooring under camera dollies), white paper vignette, LUT color grading. Explicitly reject full-screen Kuwahara — it erases the Detail pillar; at most test a distance-masked variant once.

**5. Time and seasons as data, not systems:** 4-5 LUTs per season (dawn/day/golden-hour/dusk/night, one shared 32³ format project-wide) blended by clock via URP volumes; per-season foliage colormaps + a global season lerp parameter. This makes "same hedge, four books" (Barklem's Spring/Summer/Autumn/Winter stories) an asset-swap problem a 1-3 person team can afford.

**6. Buy grass, fog, water (~$60-100 on sale); build the look.** Staggart Stylized Grass for wind + bend interaction (mice parting grass is a signature shot — test trails, particle bending, and colormap blending in week one), Atmospheric Height Fog for mouse-height ground mist + tinted distance fog, Stylized Water 3 later for the mill stream.

**7. MVP camera-scene test checklist** (one diorama: hedge bank exterior + one dollhouse interior, e.g. Crabapple Cottage kitchen):
- [ ] Camera at 35-45°, 4-8 fixed rotation steps, zoom from "whole hedge" to "teacup close-up" — ramp lighting, edge pass, and paper overlay must hold up at both zoom extremes (edge width in screen space needs distance compensation)
- [ ] 20-50k grass blades + oversized flowers with wind at 60 fps / 1080p on a GTX 1060 or Steam Deck; profile batch count with GPU Resident Drawer on vs off
- [ ] Player mouse walking through grass: bend trails, no alpha overdraw collapse at blade-level camera height
